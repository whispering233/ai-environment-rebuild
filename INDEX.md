# AI Environment Rebuild — AI 入口

本项目是一个 Markdown 驱动的 AI 环境搭建指南。AI Agent（Claude Code、Codex CLI 等）通过阅读本仓库的结构化文档，自动识别环境依赖并逐步完成安装配置。

## 项目描述

提供一套完整、可复现的 AI 工作环境搭建流程，覆盖从平台依赖（WSL2/Ubuntu）到 Agent 安装（Claude Code、Codex CLI、Aider）、运行时（Node.js/Python/Go）、配置、规则、Skills、Plugins 的全链路。

## 目录结构

| 目录 | 用途 | 分类 |
|------|------|------|
| `agents/` | AI Agent 安装知识（Claude Code, Codex CLI, Aider） | **知识层** |
| `platform/` | 平台环境依赖（WSL2, Ubuntu） | **知识层** |
| `runtime/` | 语言运行时安装（Node.js, Python, Go） | **知识层** |
| `config/` | 配置文件指引（OpenCode, Shell） | **知识层** |
| `skills/` | 推荐 Skills 列表 | **知识层** |
| `plugins/` | 推荐 Plugins 列表 | **知识层** |
| `notes/` | 注意事项和易错点 | **知识层** |
| `workflows/` | 可选工作流（按场景选择） | **编排层** |

### 知识层 vs 编排层

- **知识层**（`agents/`, `platform/`, `runtime/`, `config/`, `skills/`, `plugins/`, `notes/`）：只描述"如何安装"，不决定执行顺序。每个目录中的文件是独立的知识单元。
- **编排层**（`workflows/`）：决定"先安装什么后安装什么"。工作流按用户需求场景组织，每个工作流编排知识层中的知识单元。

## 执行流程

```
┌─────────────────────────────────────────┐
│  1. AI 读取 INDEX.md + RULES.md 理解项目 │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  2. AI 读取 workflows/README.md 选择工作流│
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  3. AI 在对话中生成安装计划              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  4. 用户审批计划                        │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  5. AI 按工作流步骤执行                 │
│     （引用各知识子目录的详细内容）        │
└─────────────────────────────────────────┘
```

## 快速开始

1. **AI 阅读者**：先阅读本文件（INDEX.md）了解全貌，然后阅读 [RULES.md](./RULES.md) 了解执行约束
2. **工作流选择**：前往 [workflows/README.md](./workflows/README.md) 按场景选择合适的工作流
3. **生成计划**：根据工作流步骤生成安装计划
4. **用户确认**：提交计划等待用户批准
5. **执行**：按批准后的计划逐步骤执行，引用对应知识子目录的详细内容
