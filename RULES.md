# AI Environment Rebuild — AI 全局规则

以下规则适用所有在此项目中工作的 AI Agent。AI 必须在执行任何操作前理解并遵守这些规则。

---

## 1. 隐私保护规则

- AI **不得**将用户的 API Key、Token、密码等敏感信息写入对话日志、文件或任何外部存储
- AI **不得**将 `local/` 目录（用户本地配置）的内容泄露到日志或版本控制
- AI **不得**上传或分享任何包含 `ANTHROPIC_API_KEY`、`OPENAI_API_KEY` 等敏感变量的文件内容
- 在生成安装计划或执行脚本时，始终使用占位符（如 `your-api-key-here`）代替真实值

## 2. 计划审批规则

- AI **必须先与用户确认计划**，获得用户明确批准后，才能执行任何安装、下载或系统修改操作
- 计划应包含：安装内容、预期变更、可能的风险
- 用户明确说"可以"或"批准"后，AI 方可执行
- 如用户未明确批准，AI **不得**执行任何修改操作

## 3. 知识/编排分离规则

- **知识层文件**（`agents/`, `platform/`, `runtime/`, `config/`, `skills/`, `plugins/`, `notes/`）只描述"如何安装"，不决定执行顺序
- **编排层文件**（`workflows/`）决定"先安装什么后安装什么"
- AI 不得直接按知识文件的目录顺序执行；必须通过工作流确定执行顺序

## 4. 工作流匹配规则

- AI **根据用户需求场景**匹配合适的工作流，而非按复杂程度选择
- 如果用户没有明确指定场景，AI 应主动询问用户的使用场景
- 工作流示例：
  - 全新环境搭建 → 选择完整安装工作流
  - 仅安装某个 Agent → 选择单 Agent 安装工作流
  - 升级/修复环境 → 选择维护工作流

---

## 违规后果

违反上述规则可能导致：
- 用户 API Key 泄露
- 环境配置损坏
- 安装流程中断
- AI 被终止执行
